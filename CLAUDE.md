## System overview

The system decodes Valve Lighthouse v2 sweep pulses from 4 TS4231 sensors on an RP2350, triangulates a 3D position, and streams MAVLink ODOMETRY (msg #331) to a Pixhawk flight controller.

### Pipeline

```
TS4231 sensors → PIO+DMA capture (core 1)
              → LFSR decode → angle EMA (core 0)
              → triangulation (solve3d)
              → MAVLink ODOMETRY @ 10 Hz → Pixhawk TELEM2
```

### Key constraints

- ArduPilot requires `MAV_FRAME_LOCAL_FRD` (20) / `MAV_FRAME_BODY_FRD` (12) in ODOMETRY — any other frame is silently discarded.
- All TS4231 PIO init must happen on core 1 (IRQs arm on the calling core).
- `bs_poses_cal.h` is generated by `utils/calibration/calibrate_export.py` — never hand-edit it.
- UF2 files in `rp2350_firmware/src/build/` are committed to git and must be rebuilt and re-committed on every source change.
